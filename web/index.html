Projekt 1 – Raspberry Pi Heimserver ▼

Aufbau eines vollständigen Heimservers auf Basis eines Raspberry Pi 4 unter Linux (Raspberry Pi OS). Der Server ist öffentlich erreichbar über DuckDNS mit Portweiterleitung über eine Fritz!Box 7560.

  • nginx als Webserver installiert und konfiguriert
  • Python REST-API entwickelt – liefert Live-Systemdaten (CPU, RAM, Temperatur)
  • nginx als Reverse Proxy eingerichtet – API über /api/status erreichbar
  • DuckDNS eingerichtet mit automatischem IP-Update via Cronjob alle 5 Minuten
  • Portweiterleitung (Port 80) in der Fritz!Box konfiguriert
  • Alle Dienste als systemd-Services eingerichtet – starten automatisch nach Neustart
  • Firewall (UFW) und Fail2ban für SSH-Schutz eingerichtet
  • SSL-Zertifikat mit Let's Encrypt – HTTPS aktiv
Raspberry Pi 4 Linux nginx Python Bash systemd DuckDNS SSH Fritz!Box REST API UFW Fail2ban Let's Encrypt
Projekt 2 – Energie-Monitoring ▼

Live-Überwachung des Stromverbrauchs des Raspberry Pi Servers. Berechnung der geschätzten Energiekosten und CO₂-Emissionen in Echtzeit – relevant für Green IT und nachhaltige IT-Infrastruktur.

  • Stromverbrauch-Schätzung basierend auf CPU Last in Python implementiert
  • Kostenberechnung basierend auf aktuellem deutschen Strompreis (0,30 €/kWh)
  • CO₂-Emissionen berechnet basierend auf deutschem Strommix
  • REST API Endpunkt /api/power erstellt
  • Gesamtstatistik wird persistent in energy_log.json gespeichert – bleibt nach Neustart erhalten
  • REST API Endpunkt /api/energy_total liefert kumulierte Werte seit Inbetriebnahme
Python REST API Green IT JavaScript Energieeffizienz JSON
Projekt 3 – IT-Helpdesk Bot ▼

KI-gestützter Helpdesk Chatbot der auf dem Raspberry Pi läuft und IT-Fragen automatisch beantwortet. Entwickelt mit Python und der Groq API (LLaMA 3).

  • Python Backend entwickelt – verarbeitet Anfragen und kommuniziert mit Groq API
  • LLaMA 3 Modell via Groq API integriert – antwortet auf IT-Fragen auf Deutsch
  • REST API Endpunkt /api/chat erstellt
  • nginx Reverse Proxy für den Bot konfiguriert
  • Als systemd-Service eingerichtet – startet automatisch
  • Frontend Chat-Interface in HTML/JavaScript entwickelt
